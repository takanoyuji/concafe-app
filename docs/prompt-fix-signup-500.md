# プロンプト: 会員登録のエラーを修正する

ブラウザの開発者ツールで出ているエラーを修正するために Claude Code に渡すプロンプトです。

---

## プロンプト（コピー用）

```
本番（https://xing-lang.com）の会員登録で、次のエラーが出ている。原因を特定して修正してほしい。

**ブラウザのコンソールに出ているエラー**
1. **500 Internal Server Error** … `POST /api/auth/signup` が 500 を返している。会員登録の送信がサーバー側で失敗している。
2. **401 Unauthorized** … `GET /api/auth/me` が 401。未ログイン時に認証状態を確認しているだけなら無視してよいが、サインアップ画面で不要に呼ばれて 401 でエラー表示になるなら、呼び方やエラーハンドリングを直してほしい。

**依頼すること**
- `/api/auth/signup` が 500 を返す原因を特定する（サーバーログの想定原因も含めて）。考えられる要因: DB（Prisma）、メール送信（lib/email.ts）、環境変数（RESEND_API_KEY や SMTP 系）、その他の例外。
- 原因に合わせてコードを修正する（環境変数未設定時のフォールバック、エラーメッセージの返し方、ログ出力など）。
- フロント（app/auth/signup/page.tsx）で、500 やネットワークエラー時に「送信中...」のままにならないように、エラー内容を表示するようにしてほしい。
- 必要なら、api/auth/me の 401 がコンソールにエラーとして出ないようにする（未ログイン時は 401 を想定内として扱うなど）。

修正後、本番で再度会員登録を試したときに 500 が出ず、エラー時はユーザーにメッセージが表示されるようにしてほしい。
```

---

## 補足

- サーバーで原因を確認するとき: `docker compose -f docker-compose.pull.yml logs -f app` で signup リクエスト時のスタックトレースを確認する。
- メール送信は `lib/email.ts` で Resend を使用。本番の .env に `RESEND_API_KEY` と `MAIL_FROM`（必要なら `NEXT_PUBLIC_BASE_URL`）が設定されているかも確認する。
